**罫線文字でASCII図を描き、文字をそのまま線に変換したSVGとして書き出すエディタ。**

公開ページ: https://miruky.github.io/keisen/

## 概要

keisenは「┌─┐」のような罫線文字の図をブラウザで書くためのエディタである。等幅のテキストエディタとパレット(細線・太線・角丸の罫線文字と矢印)、箱・表・流れ図・ツリーのひな形を備え、書いた図はリアルタイムでSVGに変換される。変換では罫線文字をフォントのグリフとしてではなく本物の線として引くため、拡大しても線の太さが揃い、フォントによって罫線が途切れる問題が起きない。線の太さと文字の間隔は書き出し設定で選べ、出力は`currentColor`で描かれるので貼り付け先のライト・ダーク配色にそのまま追従する。

書いた図は罫線文字の置換だけで一括整形できる。図全体を細線・太線へ揃える、コーナーを角丸・直角へ切り替える、外周を枠で囲う、といった操作をボタン1つで行える。書き出しはSVGのコピー・保存に加えて、SVGをそのままラスタライズしたPNGの保存にも対応する。プレビューはフィット・等倍・2倍で表示倍率を選べる。

書きかけの本文と表示設定はブラウザのlocalStorageに保存され、サーバーには何も送らない。画面はライト・ダークに対応し、右上のボタンで自動・ライト・ダークを切り替えられる。

### なぜ作ったのか

罫線文字の図はテキストファイルやコメントの中では便利だが、ドキュメントに貼ると等幅フォントの都合で線が切れたりずれたりする。一方で作図ツールで描き直すのは大げさで、元のテキストとの二重管理になる。「テキストとして書き、図として貼る」を1つのエディタで済ませ、変換後も線がきれいなままであることを目的にした。

## アーキテクチャ

![構成図](docs/architecture.svg)

変換の中心は罫線文字の表(`chars.ts`)で、各文字を「セル中央から上下左右へ伸びる線分の組」として定義する。SVG変換(`svgconv.ts`)はこの表だけを頼りに線を引き、罫線以外の文字は`text`要素として中央に置く。どちらもDOMに依存しない純粋なモジュールで、そのまま単体テストできる。

## 使い方

### 描く

エディタに直接書くか、パレットの罫線文字(細線・太線・角丸と矢印)をキャレット位置に挿入する。箱・表・流れ図・ツリーのひな形から始めて書き換えるのが早い。書き出し設定で線の太さ(細め・標準・太め)と文字の間隔(詰め・標準・広め)を選ぶと、プレビューとコピー・保存するSVG・PNGに反映される。

### 整える

「整形」の各ボタンは図全体を罫線文字の置換だけで作り直す。`太線へ`/`細線へ`は線の太さを揃え(角丸は太線化で直角の太線コーナーになる)、`角丸`/`直角`はコーナーの形を切り替える。`枠で囲う`は最も広い行に合わせた細線の枠で図全体を囲う。いずれも本文そのものを書き換えるので、結果をさらに編集できる。

### 書き出す

`SVGをコピー`・`SVGを保存`はベクタのまま、`PNGを保存`はプレビューと同じ図を現在のテーマの文字色で塗り、2倍解像度のPNGにラスタライズして保存する。SVGを表示できない場所へ貼るときはPNGを使う。キーボードからは <kbd>Ctrl</kbd>/<kbd>⌘</kbd>+<kbd>S</kbd> でSVGを保存、<kbd>Ctrl</kbd>/<kbd>⌘</kbd>+<kbd>Shift</kbd>+<kbd>C</kbd> でSVGをコピーできる。プレビューはフィット・等倍・2倍で表示倍率を切り替えられ、選んだ倍率は次回も保たれる。

### 共有する

「リンクをコピー」を押すと、いま編集している図そのものをURLのハッシュ(`#d=…`)に載せた共有リンクがコピーされる。本文はUTF-8をbase64urlにしてURLへ詰めるだけで、サーバーには何も送らない。そのリンクを開くと図が復元され、書き出し用のSVGをやり取りせずに下書きを渡せる。

### SVG変換の規則

| 入力                         | 変換                              |
| :--------------------------- | :-------------------------------- |
| 罫線文字(─│┌┐└┘├┤┬┴┼ など)   | セル中央から伸びる線(`path`)      |
| 太線の罫線文字(━┃┏┓┗┛ など)  | 太いストロークの線                |
| 角丸コーナー(╭╮╰╯)           | 直交2方向を四半円の弧でつないだ線 |
| 空白(半角・全角)             | 何も置かずに進む                  |
| その他の文字(矢印・かな漢字) | セル中央に置いた`text`            |

文字幅は全角文字を2桁、半角文字と罫線文字を1桁として数える。罫線文字は等幅フォントで1セル幅に描かれるので、エディタの見た目とSVGの座標が一致する。出力のSVGは`viewBox`と固有サイズ(`width`/`height`)を持ち、線も文字も`currentColor`で描かれる。

```svg
<svg xmlns="http://www.w3.org/2000/svg" width="30" height="60" viewBox="0 0 30 60" role="img" aria-label="罫線図">
  <g fill="currentColor" font-family="ui-monospace, 'SF Mono', Menlo, monospace" font-size="14">
  <text x="15" y="30" text-anchor="middle" dominant-baseline="central">A</text>
  </g>
  <path d="M 5 10 V 20 M 5 10 H 10 ..." stroke="currentColor" stroke-width="1.5" fill="none" stroke-linecap="square"/>
</svg>
```

### 制約

- 二重線(╔═╗)には対応しない。細線・太線・角丸と、矢印などのテキストを扱う。
- 文字幅の判定はEast Asian Widthの近似で、結合文字や絵文字幅は考慮しない。
- 持てる図は1枚だけ。複数の図を切り替える機能はない(共有リンクで別の図を持ち運ぶことはできる)。
- 共有リンクは本文をURLに含めるため、図が大きいほどURLも長くなる。

## プロジェクト構成

- `index.html` — エントリポイント
- `src/main.ts` — 起動
- `src/app.ts` — エディタ・パレット・プレビューの画面とイベント処理
- `src/icons.ts` — 線画SVGアイコン
- `src/style.css` — デザイントークンとスタイル(ライト・ダーク対応)
- `src/lib/chars.ts` — 罫線文字の線分表と文字幅の判定
- `src/lib/svgconv.ts` — テキストからSVGへの変換(線の太さ・間隔の指定込み)とcurrentColorの固定
- `src/lib/transform.ts` — 図全体の整形(太線化・角丸・枠囲い)
- `src/lib/share.ts` — 図をURLハッシュへ符号化・復号する共有リンク
- `src/lib/theme.ts` — テーマ設定(自動・ライト・ダーク)の解決
- `docs/architecture.svg` — 構成図
- `.github/workflows/` — CI(lint・テスト・ビルド)とPagesデプロイ

## 設計方針

- **グリフではなく線を引く** — 罫線文字をフォント任せで描くと、行間や書体で線が切れる。文字を線分の定義に引き直すことで、出力の見た目を入力環境から切り離した。
- **文字の表をデータとして持つ** — 罫線文字と線分の対応は1つの表で、変換ロジックは表を読むだけにした。対応文字を増やすときは表に1行足せばよい。
- **整形は本文の置換だけで行う** — 太線化・角丸・枠囲いはレイヤーや内部モデルを持たず、本文の文字列を別の文字列へ写すだけにした。結果がそのまま編集できるテキストなので、整形と手直しを行き来できる。
- **IMEを邪魔しない描画** — 本文の編集ではプレビューだけを差し替え、textareaを再生成しない。ただし整形やひな形のように本文を丸ごと入れ替えるときだけは値を差し替える。
- **貼り付け先に馴染む出力** — 線も文字も`currentColor`で描き、大きさはviewBoxに任せる。SVGを受け取った側が色と大きさを決められる。
